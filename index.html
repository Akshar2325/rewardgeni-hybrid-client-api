Hybrid Client API - Part B

Implement Point Balance and Redemption Endpoints

This documentation defines the server endpoints your HYBRID client must implement so Reward Geni can sync user points on SSO login and redeem points before order placement.

Get Started API Reference

Authentication #

Reward Geni calls your API using the following request headers:

Header Value
x-api-key Your hybridClientApiKey
Content-Type application/json

Base URL #

Reward Geni admin configures your server URL as hybridClientApiBaseUrl. Reward Geni then calls your endpoints under that base URL.

Example 
https://api.your-company.com

Point Balance Sync

Reward Geni fetches current balance blocks after SSO login.

View GET endpoint

Point Redemption

Reward Geni requests point deduction before creating an order.

View POST endpoint

Timeouts and Retry

Balance and redeem operations have defined timeout expectations.

Read retry guide

Endpoints Overview #

Method Endpoint Description
GET /v1/points/:externalUserId Get the user's point balance blocks after SSO login
POST /v1/redeem Deduct points before order creation

Error Code Reference #

Code HTTP Status When
HC_402_001 402 Insufficient points for redemption
HC_404_001 404 User not found
HC_500_001 500 Internal server error
HC_503_001 503 Service unavailable (balance or redemption)

Timeouts and Retry #

Endpoint Timeout Retries
GET /v1/points/:externalUserId 10 seconds Up to 2 retries on network errors
POST /v1/redeem 15 seconds Up to 2 retries on network errors
