Quick Start

Implement the two Part B endpoints on your server so Reward Geni can read and redeem points.

Prerequisites #

  • A server endpoint base URL that Reward Geni can call
  • Your hybridClientApiKey configured with Reward Geni
  • Support for JSON request and response payloads

Authentication #

Reward Geni includes these headers in every request:

Headers 
x-api-key: sk_live_your_api_key
Content-Type: application/json

Base URL #

Reward Geni admin stores your endpoint root as hybridClientApiBaseUrl. Example base URL:

URL 
https://your-api.example.com

Implement Required Endpoints #

  1. GET /v1/points/:externalUserId

    Return externalUserId, totalPoints, and blocks exactly as documented.

  2. POST /v1/redeem

    Deduct points and return success, transactionId, pointsRedeemed, and remainingPoints.

  3. Return standard errors

    Support 402, 404, 500, and 503 with Part B error code payloads.

First Request Example (Balance) #

cURL 
curl -X GET "https://your-api.example.com/v1/points/EMP001" \
  -H "x-api-key: sk_live_your_api_key"
JavaScript 
const response = await fetch('https://your-api.example.com/v1/points/EMP001', {
  method: 'GET',
  headers: {
    'x-api-key': 'sk_live_your_api_key'
  }
});

const data = await response.json();
console.log(data);
